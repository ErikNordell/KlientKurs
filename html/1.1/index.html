Doctype

Högst upp i alla HTML dokument ska Doctype anges. Doctype anger vilken version av HTML som hemsidan är skriven i och då hur webbläsaren ska tolka dokumentet. Sedan HTML5 räcker det att man anger detta med <!doctype html>. Medans i versioner innan HTML5 behövde man ange alla versioner som man ville stödja. När man enbart skriver <!doctype html> så väljer webbläsaren som läser dokumentet att tolka dokumentet efter den senaste versionen av HTLM.

Teckenkod

Genom att definiera vilken karaktärsuppsättning som HTLM dokumentet använder garanterar du att tecknen inte översätts eller förändras till andra tecken. Detta är viktigt för läsbarheten både för människor och datorer.

Språk

På <html> taggen bör man alltid sätta ett språk. Detta gör man genom att skriva till exempel <html lang="en"> för att sätt språket till engelska. Om det är så att man vill skriva en paragraf i ett annat språk än vas som är angivet i HTML taggen kan man ange ett nytt språk för enbart den paragrafen. Att sätta ett språk på hemsidan gör det ger mer information till till exempel sökmotorer som kan då ge bättre sökresultat.

Validering

En validerad hemsida följer vissa bestämda regler för hur en hemsida bör vara uppbyggt. Webbläsare som försöker visa ett html dokument som inte är validerad kan få behöva göra antaganden för att kunna visa innehållet i dokumentet. Detta medför att olika webbläsare kan göra olika antaganden och då visa HTML dokumentet på olika sätt.

Hur ett HTML-dokument är uppbyggt

Ett HTML dokument inleds alltid med en deklaration av vilken dokumentversion som dokumenten stödjer. Enligt HTML5 görs detta genom att <!doctype html> skrivs.
Detta följs av en html-tagg, <html> med tillhörande avslut, </html>. HTML-taggarna markerar starten och slutet av det i dokumentet som ska ses som html kod.
Mellan dessa taggar ska två andra taggar finnas, <head></head> och <body></body>. Inuti head-taggen kan det finnas en titel, stilar, script och annan information som definierar dokumentet men inte sys på när dokumentet visas. Inuti body-taggen skrivs allt som ska visas och synas för användaren, så som överskrifter, länkar, paragrafer och listor.

Metataggar

Metadata är data som inte visas för användaren men används för att beskriva din sida för datorer. Exempel på metataggar:

Author
Vem som är skaparen av webbsidan.
Description
En beskrivning av innehållet på webbsidan.
Generator
Ger information om vilket program som användes för att skapa webbsidan.
Keywords
Listar nyckelord för webbsidan.
Robots
Ger sökmotorer information om hur de ska hantera innehållet.
http-equiv
Med denna tag kan man simulera den information som finns i serverns response header utan att behöva ha tillgång till serverinställningarna.
